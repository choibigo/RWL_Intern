### 1. MSE Loss 편향(Bias)-분산(Varianve) 의 관계

- Bias-Variance Trade off
    
    $\text{Total Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Error}$

![image.png](26%2003%2012%20%EC%A7%80%EB%82%9C%EC%A3%BC%20%EC%A0%90%EA%B2%80%20%EA%B4%80%EB%A0%A8%20%EA%B3%B5%EB%B6%80/image%201.png)

- irreducible error는 문제의 본질적이 난이도에 의해 결정된다.
- 모델의 분산 → 모델이 복잡해서 생기는 오차
    - 크기가 같은 훈련 세트가 바뀔 때 발생하는 학습 성능의 변화 정도로 데이터 변동에 의한 영향을 나타낸다. 따라서 더 많은 데이터로 더 많은 학습을 수행하면 줄어들게 된다.
- 모델의 편향 → 모델이 단순해서 생기는 오차
    - 학습 알고리즘의 기대 예측 값이 실제 데이터에서 떨어진 정도이므로 학습 알고리즘의 적합 능력을 나타낸다.

적은 데이터로 훈련을 부족하게 하였을 때 학습된 모델의 적합 능력은 떨어진다. 

하지만 훈련 데이터가 조금 변경된다고 해서 학습 모델의 성능에는 큰 변화가 생기지 않는다. 이 경우에는 편향이 오차의 주된 원인이 된다.

반면, 더 많은 데이터로 훈련의 깊이를 증가시켰을 경우에는 충분히 훈련이 되어 모델의 적합 능력은 매우 좋게 된다. 반면, 훈련 데이터에 발생하는 작은 변화에도 학습기 성능은 민감하게 된다. 이는 훈련 데이터의 일반적이지 않은 특성까지 학습기가 배웠기 때문이다.

### 2. Average Pooling과 1x1 Convolution

- 특정 영역의 평균값을 뽑아낸다.
- 모델의 마지막 단계에서 **Fully Connected(FC) 레이어를 대체**하기 위해 주로 쓴다.
- 가령 $7 \times 7 \times 512$ 크기의 데이터를 $1 \times 1 \times 512$로 만들어버린다.
    
    → 파라미터 수가 획기적으로 줄어든다.
    

![image.png](26%2003%2012%20%EC%A7%80%EB%82%9C%EC%A3%BC%20%EC%A0%90%EA%B2%80%20%EA%B4%80%EB%A0%A8%20%EA%B3%B5%EB%B6%80/image%202.png)

- **1x1 Convolution과의 연관성**
    - **차원 압축:** 1x1 Conv는 가로/세로 크기는 유지하면서 **채널(깊이)의 개수**를 줄인다.
    - Inception 구조나 ResNet 같은 곳에서, 1x1 Conv로 채널을 줄여서 연산 효율을 높인 뒤, 마지막에 Average Pooling으로 공간 정보를 요약하여 최종 분류를 수행하는 식으로 함께 쓰인다.

### 3. DataLoader 클래스의 필수 요소 3가지

- 파이토치(PyTorch) 등에서 커스텀 데이터셋(`Dataset` 클래스)을 만들 때 반드시 구현해야 하는 3종 세트가 있다.
    1. **`__init__(self, ...)` (초기화):** 데이터를 어디서 가져올지 경로를 지정하고, 이미지 변형(Transform) 등의 설정을 저장하는 생성자
    2. **`__len__(self)` (길이):** 전체 데이터의 개수가 몇 개인지 숫자로 반환해야 합니다. 그래야 학습할 때 한 바퀴(1 Epoch)에 몇 번 반복할지 계산할 수 있다.
    3. **`__getitem__(self, index)` (데이터 호출):** 특정 번호(`index`)를 주면, 그에 해당하는 데이터(이미지 등)와 정답(Label)을 딱 하나 가져온다.
